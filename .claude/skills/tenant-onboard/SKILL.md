# Tenant onboarding

Onboarding a team is exactly one JSON file at `tenants/<team_name>.json`. Do not hand-write namespaces, AppProjects, or ApplicationSets — those are generated by the ArgoCD ApplicationSet git-file generator in `argocd/bootstrap/tenants/`.

## Inputs required

Ask the user (or infer from context) for:

- `team_name` — becomes the namespace, subdomain, and file name. Must be DNS-label-safe: lowercase, alphanumeric + hyphens, ≤ 63 chars, no leading/trailing hyphen.
- `sourceRepo` — the tenant's own manifest repo URL.
- `manifestPath` — path within that repo where their Kubernetes manifests (or Helm chart) live.

## Steps

1. Validate `team_name` against the DNS-label rule above. Reject and ask again if invalid.
2. Check `tenants/<team_name>.json` doesn't already exist.
3. Write the file with exactly this shape:

```json
{
  "name": "<team_name>",
  "sourceRepo": "<sourceRepo>",
  "manifestPath": "<manifestPath>"
}
```

4. Confirm the ApplicationSet generator at `argocd/bootstrap/tenants/` targets `tenants/*.json` (don't modify it unless it's broken — flag if it doesn't match).
5. If a cluster is live, dry-run the expected result: `kubectl apply --dry-run=client -f <rendered manifest>` or use the `gitops-validator` subagent.
6. Tell the user the resulting URL will be `https://<team_name>.arguswatcher.net` and the expected time-to-live (~3 min stateless, +2 min if stateful with a PVC).
